Stream is the foundation layer providing reactive programming primitives. It's a lightweight Observable implementation inspired by RxJS, but optimized for gesture handling.

## Core Concepts

### Observable

A lazy, push-based data source:

```typescript
import { createObservable } from "@cereb/core";

const numbers$ = createObservable<number>((observer) => {
  observer.next(1);
  observer.next(2);
  observer.complete();

  return () => { /* cleanup */ };
});
```

### Observer

Consumes values from an Observable:

```typescript
numbers$.subscribe({
  next: (value) => console.log(value),
  error: (err) => console.error(err),
  complete: () => console.log("done"),
});

// Or simplified:
numbers$.subscribe((value) => console.log(value));
```

### Subject

Both Observable and Observer - useful for multicasting:

```typescript
import { createSubject } from "@cereb/core";

const subject = createSubject<string>();

subject.subscribe((v) => console.log("A:", v));
subject.subscribe((v) => console.log("B:", v));

subject.next("hello"); // Both subscribers receive this
```

## Pipe and Operators

Transform streams using operators:

```typescript
import { pipe, filter, map, throttle } from "@cereb/core";

const result$ = pipe(
  source$,
  filter((x) => x > 0),
  map((x) => x * 2),
  throttle(16) // ~60fps
);
```

## Factory Functions

Create observables from common sources:

| Function | Description |
|----------|-------------|
| `fromEvent` | DOM events |
| `fromPointerEvents` | Pointer events |
| `fromTouchEvents` | Touch events |
| `fromMouseEvents` | Mouse events |
| `from` | Arrays or iterables |
| `of` | Individual values |
| `interval` | Periodic emissions |
| `timer` | Delayed emission |

See the [Stream API Reference](/api/stream/) for complete documentation.
